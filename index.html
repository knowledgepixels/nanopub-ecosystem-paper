Background and Related Work

Nanopublications a were first introduced for capturing indivual scientific claims [9,2], and later broadened to capture any kind of formally structured information entity [3,5]. Identity, verifiability, and immutability of nanopublications are addressed by Trusty URIs [8], which embed a content-derived cryptographic hash into the URI itself so that every reference and every retrieval is self-verifying. These building blocks underpin the trust model of the ecosystem presented here.

The paradigm of publishing without central gatekeepers was first outlined in [6] and subsequently realised as a running decentralized network of nanopublication servers with end-to-end provenance [4]. More recent work added a layer of decentralized services that enable semantic micro-contributions on top of that network [7]. Representative data-centric applications include the DisGeNET gene–disease resource [10] and a provenance-centered drug–drug interaction dataset [1].
